目次

1. 目標

この仕様は、 ECMAScript 仕様と, ［ ウェブブラウザの ECMAScript 実装における［ 互換性および相互運用性のための要件 ］］との相違点を，文書­化するものである。 [ECMASCRIPT] この文書の最終目標は、その内容を ECMAScript 標準に持ち上げることである。 [ECMASCRIPT] This specification aims to document the differences between the ECMAScript specification and the compatibility and interoperability requirements for ECMAScript implementations in web browsers. The ultimate goal of this document is to upstream its contents to the ECMAScript standard. [ECMASCRIPT]

【 2016 年 7 月 22 日付の改訂による結果、この仕様の実質的内容は無くなった。 詳細は、 commit 履歴 に。 】

2. 適合性

明示的に規定でないものと記された節に加え，この仕様­内のすべての 例, 注記は、規定ではない。 他のすべては規定である。 Everything in this specification is normative except for examples, notes, and sections marked as non-normative.

JavaScript 実装は、この文書に述べられる拡張を伴う ECMAScript の適合実装でなければならない。 [ECMASCRIPT] A JavaScript implementation must be a conforming implementation of ECMAScript with the extensions described in this document. [ECMASCRIPT]

ユーザエージェントはまた、適用し得る所では， HTML の適合実装でなければならない。 [HTML] A user agent must also be a conforming implementation of HTML, if applicable. [HTML]

スクリプト作者は、この仕様に定義される拡張に依存してはならない。 Script authors must not depend on the extensions defined in this specification.
