1. 序論

`CSS1$r では、指定された~sizeを伴う要素の内側に収まり切らないほど内容を配置することは、一般に 著作時の~errorとされていた。 そのような内容は、要素の境界の外側まで拡幅し，およそ 他の要素と重なり合うことになる。 ◎ In CSS Level 1 [CSS1], placing more content than would fit inside an element with a specified size was generally an authoring error. `CSS21$r では、 `overflow$p ~propが導入された。 作者は，~overflowに対し~scrollさせるよう指定できるようになり、最早 著作時の~errorでなくなった。 同時に，~overflowに対し切取るよう指定できるようにもなった — これは、作者が~overflowした内容を示したくないときに役立つ。 ◎ CSS Level 2 [CSS21] introduced the overflow property, which allows authors to have overflow be handled by scrolling, which means it is no longer an authoring error. 3.3. ~scrollingの原点, 方向, 制約

`初期~scroll位置@ とは、初期~時における — すなわち，利用者や~programによる~scroll~~操作に先立つ — ~boxの~border~boxに~~相対的な，~boxの`~scrollable~overflow領域$の位置であり、既定では，~boxの`書字mode$に依存して，~boxの`~padding辺$の［ `塊~始端$ ／ `行内~始端$ ］辺になる。 この既定は、 `CSS-ALIGN-3$r の［ `align-content$p, `justify-content$p ］~propを利用して変更することもできる。 ◎ The initial scroll position, that is, the initial position of the box’s scrollable overflow region with respect to its border box, prior to any user or programmatic scrolling that changes it, is dependent on the box’s writing mode, and is by default the block-start/inline-start edge of the box’s padding edge. However, the align-content and justify-content properties [CSS-ALIGN-3] can be used to change this.

~web互換性をとるための拘束に因り（これは、旧来の~bugにつけ込んで，視覚系~browserに対しては こっそり内容を隠しながら, 検索engineや発話~出力に対してはそうしなかった，作者たちに~~起因するものである）、~UAは，`~scroll容器$の`~scrollable~overflow領域$を，その~boxの［ `塊~始端$, `行内~始端$ ］側で切取ることが要求される（したがって，それらの側には`~scrollable~overflow$はなかったかのように挙動することになる）。 ◎ Due to Web-compatibility constraints (caused by authors exploiting legacy bugs to surreptitiously hide content from visual readers but not search engines and/or speech output), UAs must clip the scrollable overflow region of scroll containers on the block-start and inline-start sides of the box (thereby behaving as if they had no scrollable overflow on that side).

表示域は、これらの計算に`原則的な書字mode$を利用する。 ◎ The viewport uses the principal writing mode for these calculations.

5. 静的~媒体における~overflow

この仕様は、静的~媒体（印刷など）における `overflow$p のすべての値に対し、有用な挙動を定義するべきである。 現在の実装による挙動は、ごく拙いものであり、作者が 対話的~媒体に対し生産される内容が印刷されるときに何が起こるか考慮していない場合には，期待されない結果を生産している。 ◎ This specification should define useful behavior for all values of overflow in static media (such as print). Current implementation behavior is quite poor and produces unexpected results when authors have not considered what will happen when the content they produce for interactive media is printed.

6. ~privacyと~security上の考慮点

この仕様は、新たな~privacyや~security上の懸念を導入するものではない。 ◎ This specification introduces no new privacy or security concerns.
