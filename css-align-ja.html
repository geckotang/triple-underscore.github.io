~INFORMATIVE

3. 整列の各種用語

この~moduleは、 CSS におけるすべての~layout~modeに対する各種 整列~propを定義するので、抽象的な各種用語を導入する： ◎ Since this module defines alignment properties for all layout modes in CSS, some abstract terminology is introduced:

`整列~対象0@
`整列~対象0$とは、 ~propにより整列される もの／ものたち である。 `justify-self$p ／ `align-self$p に対する`整列~対象0$は、~propが設定されている~boxの`~margin~box$になる。 `justify-content$p ／ `align-content$p に対する`整列~対象0$は、各~layout~modeごとに定義される。 ◎ The alignment subject is the thing or things being aligned by the property. `整列~容器@
`整列~容器$は、その中で`整列~対象0$が整列される矩形である。 これは，各~layout~modeごとに定義されるが、通例的には，`整列~対象0$の`包含塊$になる。 ◎ The alignment container is the rectangle that the alignment subject is aligned within. This is defined by the layout mode, but is usually the alignment subject’s containing block.
`~fallback整列@
整列によっては、ある種の状況に限って充足されたり，消費できる空間が制限されることもある。 例えば， `space-between$v は、`整列~対象0$が 1 個以上ある場合に限り演算し得る。 また，基底線~整列は、充足された時点で，余った空間すべてを吸収するには不十分かもしれない。 これらの事例では`~fallback整列$が（下に定義されるように）効果を~~発揮し，余った空間を全部的に消費する。 ◎ Some alignments can only be fulfilled in certain situations or are limited in how much space they can consume; for example, space-between can only operate when there is more than one alignment subject, and baseline alignment, once fulfilled, might not be enough to absorb all the excess space. 8.2. 基底線~整列の各種用語

  • それらの基底線すべてが、互いに平行な`行内~軸$~沿いの下で，`整列~文脈を共有-$している。
  • それらの`塊~flow方向$は、互いに同じ向きである（互いに平行であるのみならず）。

  • ~tableの~col軸（塊~軸）沿いの，同じ~col内の~table~cellたち ◎ table cells in the same column, along the table’s column (block) axis
  • 格子の~row軸（行内~軸）沿いの，同じ~row内の`格子~駒$たち ◎ grid items in the same row, along the grid’s row (inline) axis
  • 格子の~col軸（塊~軸）沿いの，同じ~col内の`格子~駒$たち ◎ grid items in the same column, along the grid’s colum (block) axis
  • `~flex容器$の主~軸~沿いの，同じ`~flex行1$内の`~flex駒$たち ◎ flex items in the same flex line, along the flex container’s main axis
