この仕様は、~textおよび関係する入力に対する追加の~eventを定義する。 この仕様は、 `UIEVENTS$r の上に築かれている。 ◎ This specification defines additions to events for text and related input. This specification builds on the Document Object Model Events Level 3.

目次

    1. 序論

    この文書は、 `UIEVENTS$r にて述べられれた 2 つの~event — `input$et, `beforeinput$et — に対する，編集に関係する追加について述べる。 これらの~eventの目標は、作者が編集-時の既定の挙動を，編集が生じる前と後の両者について，理解したり上書きできるようにすることである。 ◎ This document describes editing related additions to 2 events - Input and beforeinput which are described in the UI events spec. The goal of these events is to allow authors to understand and/or override default edit behavior both before and after editing occurs.

    2. 問題と利用事例

    この文書が取り組む問題と利用事例については、 `Editing-Explainer$r を~~参照されたし。 ◎ The problems solved by and use cases for this document by this document are tracked in the Editing Explainer document.

    3. 入力~event型

    入力~eventは、利用者が~markupを編集しようと試みる前（ `beforeinput$et ~event ）と試みた後（ `input$et ~event）に送信される。 これには、 文字, 削除, その他の関係する編集-が含まれる。 ◎ Input events are sent before (beforeinput event) and after (input event) a user attempts to edit the markup. This includes characters, deletions, and other related edits.

    3.2. 各種~入力~event型

    以下に入力~event型を挙げる。 ◎ The Input event types are listed below.

    `●型^ `beforeinput@et `●界面^ `InputEvent$I `●同期^ あり `●浮上^ する `●標的^ `contenteditable$a 属性が可能化されている任意の `Element$I ◎ Any Element with contenteditable attribute enabled. `●取消^ 文脈依存：
    • ~IMEによる組成の間であって， `inputType$m が ［ `insertText^l ／ `deleteComposedCharacter^l ］ならば ~F
    • 他の場合は ~T
    ◎ Varies: False for inputTypes insertText and deleteComposedCharacter during IME composition. True otherwise `●既定動作^ 文脈依存：
    • `contentEditable$m が `typing^l にされた`編集中の~host$においては、 `inputType$m が［ `insertText^l ／ `deleteComposedCharacter^l ］ならば， “DOM を更新する” 。
    • `contentEditable$m が `true^l にされた`編集中の~host$においては、 `inputType$m が何であれ， “DOM 要素を更新する”
    • 他の場合は、なし
    ◎ Varies: 'Update the DOM' for contentEditable=typing editing hosts for inputTypes insertText and deleteComposedCharacter. 'Update the DOM element' for contentEditable=true editing hosts for all inputTypes. None otherwise. `●文脈^
    `Event.target$m
    更新されつつある `~event標的$ ◎ Event.target: event target that is about to be updated
    `UIEvent.view$m
    `Window$I ◎ UIEvent.view: Window
    `UIEvent.detail$m
    0 ◎ UIEvent.detail: 0
    `InputEvent.data$m
    要素に追加される~dataを包含している文字列。 内容が削除された場合は`空~文字列$になる。 適用されない場合は ~NULL になる。 ◎ InputEvent.data: the string containing the data that will be added to the element, which MAY be the empty string if the content has been deleted, or NULL if it doesn't apply.
    `InputEvent.isComposing$m
    この~eventが［ `~dead-key$連列の間 ／ `~IME$が作動中の間（`組成~event$が配送されているなど） ］に配送されたならば ~T 。 他の場合は ~F 。 ◎ InputEvent.isComposing: true if this event is dispatched during a dead key sequence or while an input method editor is active (such that composition events are being dispatched); false otherwise.
    `Event.cancelable$m
    この~eventを取消せるならば ~T ／ 他の場合は ~F ◎ InputEvent.cancelable: true if this event can be canceled. false otherwise.
    すべての `beforeinput$et ~eventは、~IMEによる組成の間を除き，取消可能で~MUST。 ◎ All beforeinput events MUST be cancelable, unless they take place during IME composition.
    `●表終^

    ~UAは、利用者が `contenteditable$a 要素~内で入力を試みたときに，この~eventを配送し~MUST。 これは、~UAが DOM を更新することを意味するとは限らない。 ◎ A user agent MUST dispatch this event when the user has attempted to input in a contenteditable element. It does not necessarily mean the user agent will then update the DOM.

    ~UAは、~system~eventなど，利用者により試みられた入力~以外に因る~eventに対しては、この~eventを配送しては~MUST_NOT。 ◎ A user agent MUST NOT dispatch this event due to events that are not caused by attempted user input, such as system events.

    `●型^ `input@et `●界面^ `InputEvent$I `●同期^ あり `●浮上^ する `●標的^ `contenteditable$a 属性が可能化されている任意の `Element$I ◎ Any Element with contenteditable attribute enabled. `●取消^ 不可 `●既定動作^ なし `●文脈^
    `Event.target$m
    更新されたばかりの`~event標的$ ◎ Event.target: event target that was just updated
    `UIEvent.view$m
    `beforeinput$et と同様 ◎ UIEvent.view: Window ◎ UIEvent.detail: 0
    `InputEvent.data$m
    要素に追加された~dataを包含している文字列。 内容が削除された場合は`空~文字列$になる。 適用されない場合は ~NULL になる。 ◎ InputEvent.data: the string containing the data that was added to the element, which MAY be the empty string if the content has been deleted, or NULL if it doesn't apply.

    `beforeinput$et と同様 ◎ InputEvent.isComposing: true if this event is dispatched during a dead key sequence or while an input method editor is active (such that composition events are being dispatched); false otherwise.
    `Event.cancelable$m
    ~F ◎ InputEvent.cancelable: false
