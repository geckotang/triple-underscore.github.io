1. 序論

~INFORMATIVE

伝統的に、一部の~UA（例： Chrome, Safari, Edge ）は、（非首~buttonも含め）すべての~buttonに対し~click~eventを発火してきた。 これは、~click~event~handlerを実行している頁にて，開発者が非首~buttonが用いられる局面を考慮してなかったときに、いくつかの課題をもたらす（例えば，中央~clickで 新たな~tab内に~linkを開くときに失敗するのは、これらの~browser利用者にとって共通的な不満の種になっている）。 他の~UAは、ここでは異なる~approachに従っていた。 Firefox は、非首~buttonによる~click~eventを，文書~levelの~nodeのみに配送する。 ◎ Traditionally some users agents (eg. Chrome, Safari and Edge) have fired the click event for all buttons (including non-primary). This poses a number of issues with pages executing click event handlers where the developer hasn't considered non-primary-button scenarios (for example, middle-click failing to open links in a new tab is a commmon source of user complaints in these browsers). Other user agents have followed different a approaches here. Firefox dispatches non-primary-button clicks event only to the document level nodes.

既存の `click^et ~handlerに対する互換性を最大にするため（それは、非首~buttonによる局面を ほぼ考慮しない）、 `click$et ~eventは、首~buttonに限り発火される。 しかしながら、~UAによる非首~button~clickに対する既定の挙動を，依然として上書きしたいと望む~appも中にはある。 特に中央~buttonに対する~event, および 既定の挙動を明示的に定義することが望まれる。 ◎ For maximum compatibility with existing click handlers (which mostly do not consider non-primary button scenarios), the click event is defined to fire only for the primary button. However some applications may still wish to override the default behavior of the UA for non-primary button clicks. Specifically it is desirable to have an event for the middle button and explicitly define the default behavior.
