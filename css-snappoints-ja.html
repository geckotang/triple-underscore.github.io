    ~INFORMATIVE

    ~scroll可能な内容に対する UX ~paradigmでは、頁割り†など，内容を論理的な何~節かに区分するものに人気があり、多用されている。 これはとりわけ、~touch操作による内容とのやりとりに該当する — 利用者にとっては、~tap~navigationを通して階層~構造を掘下げるより，平坦に拡がるように配列された内容を~panする方が、容易かつ素早く操作できるので。 例えば，写真~集の中の多数の写真を閲覧するとき、個々の写真を~tapするより 写真の~slideshowを~panする††方が，利用者には容易い。 ◎ Popular UX paradigms for scrollable content frequently employ paging through content, or sectioning into logical divisions.     ◎ Snap positions can be specified as a particular alignment (scroll-snap-align) of an element’s scroll snap area (scroll-snap-margin, defaulting to its border box) within the scroll container’s snapport (its scrollport, as reduced by its scroll-snap-padding).     ~scroll容器の~scrollportの~scroll位置を留め位置に整列されるように調整する動作は、 `留め@ （ snap ／ snapping ） と呼ばれる。 `~scroll容器$は、その~scrollportの~scroll位置がある`留め位置$にあって，作動中の~scroll操作はないとき、その留め位置に`留められ$ているという。 この~moduleは、`留め$の施行-時に利用する精確な~animationや物理的挙動については，何も 指定しないし義務付けない — それは、意図的に~UAに委ねられる。 ◎ The act of adjusting the scroll position of a scroll container’s scrollport such that it is aligned to a snap position is called snapping, and a scroll container is said to be snapped to a snap position if its scrollport’s scroll position is that snap position and there is no active scrolling operation.     6.3. 軸留め vs 点留め

    この特色機能は、 Level 1 の特色機能を~~絞るため，次回の~~発行では除去されることが計画されている。 これは、将来の Level 2 の基準を定義するため，ここに含められている。 ◎ This feature is planned to be removed in the next publication in order to reduce the feature-set of Level 1. It is included here for future reference in defining Level 2.

    `~scroll容器$が従事する `留め時の挙動@ には、次の 2 つがあり得る： ◎ There are two distinct snapping behaviors that a scroll container might engage in:

    `軸留め@ ◎ axis-snapping
    `軸留め$する`~scroll容器$では、その各~子孫は，欲される~scroll位置を，`~scroll容器$の各~軸ごとに独立に指示する。 一方の軸における~scroll位置の選好が，他方のそれに関わることはない。 ◎ If a scroll container is axis-snapping, its descendants indicate a desired scroll position in each axis of the scroll container independently, with no dependent preference for what the other axis’s scroll position should be.
    注記： これは、ほとんどの`~scroll容器$に求められる “既定の” `留め時の挙動$なので、 `scroll-snap-type$p ~propの既定は，意図的にこの挙動にされている。 ◎ Note: This is the “default” type of snap behavior that most scroll containers will want to use, and so the scroll-snap-type property intentionally defaults to it.
    注記： `軸留め$している`~scroll容器$ 内の要素は、各~軸ごとに`留め位置$を宣言できる。 一方の軸にて，ある要素の`留め位置$が選ばれても、他方の軸の`留め位置$には無関係になる — 他方の軸では、同じ, あるいは別の要素の`留め位置$が選ばれるかもしれないし，全く留められないかもしれない。 ◎ Note: An element in an axis-snapping scroll container can declare two snap positions, one in each axis. If one of the element’s snap positions is chosen in one axis, this has no bearing on the other dimension’s snap position—it might be chosen, or a different element’s snap position might be chosen for that axis, or that axis might not snap at all.
    `点留め@ ◎ point-snapping
    `点留め$している`~scroll容器$では、その各 子孫は，欲される~scroll位置を，`~scroll容器$の両~軸とも同時に指示する — 言い換えれば、子孫~内のある地点が，`~scroll容器$内の対応する地点に整列され~MUST。 ◎ If a scroll container is point-snapping, its descendants indicate a desired scroll position in both axes of the scroll container simultaneously—in other words, some point in the descendant must be aligned to a corresponding point in the scroll container.
    この挙動は、 “二次元の” ~pan操作~型の~layoutに意図されている。 例えば、地図~上の都市（ 2D 留め位置に `proximity$v を利用して，表示の中心に都市が近づいたとき，そこに留める），あるいは 画像の敷き詰め展示（ 2D 留め位置に `mandatory$v を利用して，各~画像を~screenの中心に強制する）。 これら両~事例とも、~scroll時に 縦, 横 それぞれの~~方向で異なる要素に整列されたなら（`軸留め$する場合に得られる挙動），不可解なふるまいになる。 ◎ This type of snapping behavior is intended for "two-dimensional" panning-type layouts, such as cities on a map (using proximity 2D snap positions to snap a city to the center of the display when it gets close), or a tiled image gallery (using mandatory 2D snap positions to force each image to be centered on the screen). In both of these cases, it would look weird if the horizontal scrolling was aligned to one element while the vertical was aligned to a different element (which is the behavior you’d get if the scroll container was axis-snapping).

    7. ~privacyと~security上の考慮点

    この仕様は、すでに直接的に DOM に公開されているものを超えるような，いかなる情報も公開させない。 これは、~scroll~~操作を少しばかり機能的にするだけである。 新たな ~privacy／~security 上の考慮点はない。 ◎ This specification does not expose any information whatsoever that is not already exposed to the DOM directly; it just makes scrolling slightly more functional.
